# Security Policy

## Reporting a Vulnerability

The Togetherwise team takes security seriously. We appreciate your efforts to responsibly disclose any vulnerabilities you discover.

### How to Report

If you believe you have found a security vulnerability in this project, please report it privately through one of the following channels:

- **Email:** [shamalsathsara4@gmail.com](mailto:shamalsathsara4@gmail.com)
- **Phone:** +94 72 357 7218

> **⚠️ Please do not open a public GitHub issue for security vulnerabilities.**

### What to Include

To help us triage and respond quickly, please include as much of the following as possible:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact or severity
- Any suggested fixes or mitigations (if applicable)

### What to Expect

| Stage | Timeframe |
| --- | --- |
| Acknowledgement of your report | Within **48 hours** |
| Initial assessment & triage | Within **5 business days** |
| Resolution or mitigation plan | Within **30 days** |

We will keep you informed throughout the process. If the vulnerability is confirmed, we will:

1. Work on a fix and coordinate a release timeline
2. Credit you in the release notes (unless you prefer to remain anonymous)
3. Notify you when the fix has been deployed

### Scope

This policy applies to the latest version of the Togetherwise platform, including:

- Frontend client (`/client`)
- Backend server (`/server`)
- API endpoints and authentication flows
- Database interactions

### Safe Harbor

We consider security research conducted in accordance with this policy to be authorized and will not pursue legal action against researchers who report vulnerabilities responsibly.

Thank you for helping keep Togetherwise and its community safe. 💚
